# 모라브(Morav) - PRD (제품 요구사항 정의서)

## 프로젝트 개요
- **프로젝트명**: 모라브 (Morav)
- **한 줄 요약**: 블로거들을 위한 다중 블로그 통합 발행 솔루션
- **세션 날짜**: 2026-01-08
- **문서 버전**: v1.0

---

## 1. 문제 정의 (Problem Statement)

현재 애드센스 수익을 노리는 다중 블로그 운영자들은 다음과 같은 고충을 겪고 있습니다:

- **트렌드 추적의 어려움**: 실시간으로 변화하는 인기 검색어를 수동으로 모니터링하고 포착하는 것은 시간 소모적이며 기회를 놓치기 쉽습니다
- **콘텐츠 생성 병목**: AI로 콘텐츠를 생성하더라도 각 블로그 플랫폼(티스토리, 구글 블로그, 워드프레스)에 개별 로그인하여 수동 발행해야 하는 번거로움
- **확장성 한계**: 3개 이상의 블로그를 동시 운영할 때 관리 복잡도가 기하급수적으로 증가하여 실질적인 수익 확대가 어려움
- **카테고리 불일치**: 범용 트렌드 키워드가 본인 블로그 주제와 맞지 않아 무용지물이 되는 경우가 빈번함

모라브는 **인기 검색어 자동 추출 → AI 콘텐츠 생성 → 다중 플랫폼 자동 발행**의 완전 자동화 파이프라인을 제공하여, 블로거가 설정만 해두면 손 안 대고도 지속적인 트래픽 유입을 확보할 수 있게 합니다.

## 2. 목표 및 성공 지표 (Goals & Success Metrics)

### 비즈니스 목표
- **1차 목표**: 런칭 후 6개월 내 유료 구독자 100명 확보
- **궁극 목표**: 애드센스 블로거들의 트래픽 유입을 극대화하여 월 수익 증대에 기여

### 성공 지표 (Success Metrics)
**결과 지표 (Outcome Metrics)**
- 유료 구독자 100명 달성
- 사용자 평균 블로그 유입수 전월 대비 +50% 이상

**입력 지표 (Input Metrics)**
- 무료 체험(5건) 완료율 70% 이상
- 무료 → 유료 전환율 20% 이상
- 일평균 자동 발행 콘텐츠 500건 이상 (전체 사용자 합산)
- 사용자 재방문율 (주 3회 이상) 60% 이상

## 3. 사용자 페르소나 (User Persona)

### 주요 페르소나: "부업 블로거 민수"
- **연령/직업**: 30대 중반, IT 회사 직장인
- **목표**: 애드센스를 통해 월 100만원 부수입 달성
- **행동 패턴**: 퇴근 후 저녁 시간과 주말에 블로그 관리
- **보유 블로그**: 티스토리 2개, 워드프레스 1개 운영 중
- **통증점 (Pain Points)**:
  - 트렌드 키워드를 놓쳐서 타이밍을 못 잡는 경우가 많음
  - 같은 콘텐츠를 3개 블로그에 복붙하는 데만 30분 소요
  - AI 글은 있는데 발행이 병목이 되어 수익 확장이 안 됨
- **니즈**: 한 번 설정해두면 자동으로 돌아가는 시스템, 결과만 확인하고 싶음

## 4. 사용자 스토리 (User Stories)

### 핵심 기능별 사용자 스토리

**FEAT-1: 인기 키워드 자동 수집 & 카테고리 필터링**
- **AS** 애드센스 블로거
- **I WANT** 내 블로그 카테고리(예: 건강, IT, 육아)에 맞는 실시간 인기 검색어만 자동 수집되길 원함
- **SO THAT** 내 블로그 주제와 무관한 키워드로 콘텐츠를 낭비하지 않고, 유입 가능성이 높은 글만 발행할 수 있음

**FEAT-2: AI 콘텐츠 자동 생성 (사용자 BYOK)**
- **AS** 애드센스 블로거
- **I WANT** 내가 보유한 AI API(OpenAI, Claude, Gemini, Grok)로 키워드 기반 콘텐츠(텍스트+이미지)가 자동 생성되길 원함
- **SO THAT** API 비용을 직접 관리하면서도 애드센스 승인 가능한 품질의 콘텐츠를 확보할 수 있음

**FEAT-3: 다중 플랫폼 자동 발행 & 예약 스케줄링**
- **AS** 애드센스 블로거
- **I WANT** 티스토리, 구글 블로그, 워드프레스 중 내가 선택한 플랫폼에 원하는 시간대로 자동 발행되길 원함
- **SO THAT** 수동 로그인 없이 완전 자동화된 상태로 지속적인 콘텐츠 유입을 확보할 수 있음

## 5. MVP 범위 (Minimum Viable Product)

### MVP에 포함되는 기능 (In Scope)
1. **회원 가입 & 로그인**: 이메일 기반 회원가입, OAuth 소셜 로그인 (선택)
2. **블로그 계정 연동**: 티스토리, 구글 블로그, 워드프레스 OAuth 연동
3. **AI API 키 등록**: 사용자 본인 소유 OpenAI/Claude/Gemini/Grok API 키 암호화 저장
4. **카테고리 설정**: 사용자 블로그 주제에 맞는 키워드 카테고리 선택 (예: 건강, IT, 육아)
5. **인기 키워드 자동 수집**: 네이버/구글 실시간 트렌드 키워드 크롤링 및 DB 저장
6. **AI 콘텐츠 자동 생성**: 키워드 기반으로 텍스트 + 이미지 자동 생성
7. **발행 시간 예약**: 사용자가 원하는 시간대에 자동 발행되도록 스케줄 설정
8. **다중 플랫폼 자동 발행**: 선택한 블로그에 동시 발행
9. **대시보드**: 발행 현황, 남은 발행 건수, 구독 플랜 정보 확인
10. **결제 시스템**: 토스페이먼츠/스트라이프 연동, 구독 관리

### MVP에서 제외되는 기능 (Out of Scope)
- 네이버 블로그 연동 (향후 확장)
- SEO 최적화 자동 분석 (향후 확장)
- 애드센스 수익 리포트 통합 (향후 확장)
- 콘텐츠 수정/편집 기능 (v1은 자동 발행만)
- 팀 협업 기능 (v1은 개인 사용자만)

## 6. 비기능적 요구사항 (Non-Goals)

- **콘텐츠 책임**: 사용자가 등록한 AI API로 생성된 콘텐츠의 저작권 및 법적 책임은 사용자에게 있음
- **플랫폼 정책 준수**: 각 블로그 플랫폼(티스토리, 구글, 워드프레스)의 자동 발행 정책은 사용자가 확인해야 함
- **AI API 비용**: 사용자 본인 API 사용량에 따른 과금은 모라브가 책임지지 않음

## 7. 가정 및 리스크 (Assumptions & Risks)

### 가정 (Assumptions)
1. 타겟 사용자는 AI API 키를 직접 발급받고 관리할 능력이 있음
2. 티스토리, 구글 블로그, 워드프레스 API가 안정적으로 유지됨
3. 애드센스 승인 기준을 만족하는 AI 콘텐츠 생성이 가능함

### 리스크 (Risks)
| 리스크 | 영향도 | 완화 전략 | 트리거 | 대응 |
|--------|--------|-----------|--------|------|
| 플랫폼 API 정책 변경 | 높음 | 공식 API 사용, 정기 모니터링 | API 에러율 급증 | 대체 플랫폼 추가 또는 사용자 알림 |
| 키워드 수집 서버 비용 폭증 | 중간 | 캐싱, DB 최적화, 요금제 조정 | 월 서버 비용 예산 초과 | 키워드 수집 빈도 조정 |
| AI 콘텐츠 품질 저하 (애드센스 탈락) | 높음 | 프롬프트 최적화, 품질 가이드 제공 | 사용자 불만 증가 | AI 모델 업그레이드 가이드 제공 |
| 무료 → 유료 전환율 저조 | 중간 | 5건 체험 최적화, 온보딩 개선 | 전환율 10% 미만 | 무료 건수 확대 또는 가격 조정 |
| 경쟁 서비스 출현 | 중간 | 빠른 기능 개선, 차별화 강화 | 시장 조사 | 추가 기능 개발 |

## 8. 해결 가설 (Solution Hypothesis)

**우리는 다음과 같이 믿습니다:**

- 애드센스 블로거들은 **트렌드 키워드 추적 + AI 콘텐츠 생성 + 다중 발행**의 3단계를 모두 자동화하면 월 19,000원~199,000원을 기꺼이 지불할 것입니다
- 사용자가 본인의 AI API를 직접 관리하는 BYOK 모델은 비용 투명성과 통제권을 제공하여 신뢰도를 높일 것입니다
- 카테고리별 키워드 필터링 기능은 무분별한 콘텐츠 생성을 방지하고 실질적인 트래픽 증가로 이어질 것입니다
- 5건 무료 체험은 사용자가 실제 효과를 경험하게 하여 20% 이상의 전환율을 달성할 것입니다

**이를 검증하기 위해:**
- MVP 런칭 후 첫 3개월간 사용자 행동 데이터(체험 완료율, 전환율, 발행 패턴)를 추적하고
- 사용자 인터뷰를 통해 실제 트래픽 증가 여부와 가격 만족도를 확인합니다

## 10. 실험-학습 클로저 (Experiment-Learning Loop)

**초기 가설 → 실험 → 관측 → 학습 → 다음 가설**

**주기 1 (런칭 후 1개월)**
- **실험**: 5건 무료 체험 제공, 전환율 측정
- **관측**: 체험 완료율, 유료 전환율, 이탈 구간
- **학습**: 어느 단계에서 이탈이 많은지 파악
- **다음**: 온보딩 개선 또는 무료 건수 조정

**주기 2 (2~3개월)**
- **실험**: 카테고리별 키워드 필터 효과 검증
- **관측**: 사용자가 설정한 카테고리 vs 실제 발행 패턴
- **학습**: 어떤 카테고리가 가장 인기 있는지
- **다음**: 인기 카테고리 템플릿 제공
